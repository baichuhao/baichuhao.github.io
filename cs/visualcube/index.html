Visual Cube教程

Visual Cube是一款虚拟魔方工具，但是这个是不能动的，所以，很多人用Visual Cube来制作魔方教程，这样可以让你的魔方教程更加生动形象，让你的魔方教程的公式都有一个图，或者说像下面图二图三一样，画一个花式魔方图。 这个是完成了的，给大家看看它可以做些什么

图一生成方式：
http://cube.crider.co.uk/visualcube.php?bg=t&fmt=png&view=plan&arw=U3U5,U5U7,U7U3&ac=r

图二生成方式：
http://cube.crider.co.uk/visualcube.php?bg=t&fmt=png&alg=M2 S2 E2

图三生成方式：
http://cube.crider.co.uk/visualcube.php?fmt=svg&size=200&pzl=7&cc=grey&r=z-15x-105&sch=yyyyyy&fc=ynyyynynnnynnnyyyyyyyyynnnyynyynyynnnyyynnynnnnny

VisualCube Editor是一位魔友有一个日本魔友根据visualcube的源网页，做了一个图形用户界面（GUI），对新手就友好很多。用Visual Cube的内核创造的VisualCube Editor。VisualCube Editor方法是一种简单易懂的方法，在填写完下面的参数过后点击“submit”即可生成图片，然后将图片保存即可。新人友好。VisualCube Editor

VisualCube GUI是我制作的一个VisualCube的编辑器，相比VisualCube Editor，VisualCube GUI包含了VisualCube的所有功能，而且对于新人更友好。VisualCube GUI

网址方法类似于编程，但它又异于编程，Visual Cube是一个服务器程序，在网址框中添加前缀“http://cube.cirder.co.uk/visualcube.php?”即可（“.”参数除外），然后后面写上各种参数，每个参数后面都要打一个“&”符号，表示这个参数填写完成，填写下一个参数。参数和这个参数的数值用“=”符号连接。
格式：http://cube.cirder.co.uk/visualcube.php?参数一:数值&参数二:数值&参数三:数值&参数四:数值&参数五:数值......
例子：http://cube.crider.co.uk/visualcube.php?fmt=png&view=plan&pzl=3&bg=t
*注意：你的网址必须全部是英文，参数也是。

Visual Cube中参数是最重要的，参数用来告诉Visual Cube要生成什么样的魔方。

没错，这个就是上面的那个点，如果你使用编辑器方法，那么可以不用看，如果你是使用网址方法，这个可是一个能让你加快制作教程速度的东西，使用这个方法的话，上面的格式就不管用了，不过也是大同小异，把http://cube.cirder.co.uk/visualcube.php?的/visualcube.php?就要改成/visualcube.png?（不一定是png，png / gif / jpg / svg / tiff / ico都可以），默认svg格式。
默认值：.svg
网址方法格式：http://cube.cirder.co.uk/visualcube.svg | .gif | .jpg | .svg | .tiff | .ico ?
例子：http://cube.cirder.co.uk/visualcube.svg?
这个会生成一个格式为svg的魔方图片

fmt这个参数的意思是图片格式，Visual Cube怎么会知道你要生成什么格式的图片呢？所以要告诉它，在编辑器方法中是第三个框的“Image Format”，是一个下拉菜单，点击过后选择你要的图片格式即可，做教程建议使用svg格式的，做软件图标建议用ico格式的（在编辑器方法中为icon），其他格式自己酌情考虑。可以填写png / gif / jpg / svg / tiff / ico这几种格式。默认svg格式
默认值：.svg
网址方法格式：fmt=png | gif | jpg | svg | tiff | ico
例子：http://cube.crider.co.uk/visualcube.php?bg=t&fmt=svg
这个也会生成一个格式为svg的魔方图片
PS：这个参数二可以说是参数一的“标准版”，因为你会发现，在你用了参数一的时候，网址会自动变成参数二的形式。另外建议每一个图片都弄上背景，背景后面会讲，一般不是svg，gif格式的不弄背景都无法显示魔方。（这就是以前很多人说png格式的失效了的原因）

pzl是用来告诉Visual Cube要生成图片中魔方的阶数，目前仅支持1-10阶，不可能你要弄一个超高阶的教程吧，弄花式的可能会被累死，这个可以说已经完全够用，因为wca比赛的正阶就是2-7阶。在编辑器方法中这个选项在第四栏，同样是一个下拉菜单，点击过后选择其中的一项即可。默认三阶。
默认值：3
网址方法格式：pzl=1 - 10
例子：http://cube.crider.co.uk/visualcube.php?pzl=7&bg=t&fmt=svg
这个将会生成一个七阶魔方图片

注意：大于4阶的魔方不能用case/alg参数，如果使用，只能旋转最外层的面。

所以，可见这个参数不填的时候，生成的魔方是一个有底色的魔方，在制作f2l之类的公式时建议使用。
当填写这个参数，且将数值填写为”plan”时，会生成一个魔方的俯视图，会显示你现在魔方的U层，适合用来做oll，pll，coll之类的公式。
当填写这个参数，且将数值填写为”trans”时，生成的魔方底色设置为透明，并隐藏任何蒙版或未定义的面。适合f2l，cross之类的公式。
这个参数在编辑器方法中是第六个框，也是一个下拉菜单，选择“-”相当于不填，剩下两个如果要使用，就选择其中一个就好了。
默认值：无
网址方法格式：view=plan|trans（视图）
例子1：http://cube.crider.co.uk/visualcube.php?fmt=png&view=plan&bg=t
例子2：http://cube.crider.co.uk/visualcube.php?fmt=png&view=trans&bg=t

这个参数是一个较为重要的参数，我们先来看一下它能生成什么样的魔方图片：

所以，你会发现，在你制作oll，f2l之类的公式时，不得不用这个参数，这个参数的作用就是给魔方加遮罩，支持fl / f2l / ll / cll / ell / oll / ocll / oell / coll / ocell / wv / vh / els / cls / cmll / cross / f2l_3 / f2l_2 / f2l_sm / f2l_1 / f2b / line / 2x2x2 / 2x2x3。 fl：魔方D层，其余遮盖（目前仅对三阶二阶有效）

f2l：魔方前两层，相信魔友们都很熟悉（目前仅对三阶有效）
ll: 魔方顶层，其余遮盖（目前仅对三阶二阶有效）
cll: 说是cll，竟然对二阶无效！仅支持三阶，显示顶层角块及顶层中心块
ell:顶层棱块加顶层中心块（不就是顶层的一个十字吗？）（目前仅对三阶有效）
oll：仅显示U面，做oll用，但是在不同的魔方有不同的效果（仅二三阶），如果运用到三阶，那么就是显示oll，如果用在二阶，那么就是一个制作色先公式的料啦（会显示U,D面颜色）
ocll：cll附加oll，顶面角块和顶面中心块颜色（同cll，仅支持三阶）
oell:ell附加oll，顶面棱块和顶面中心块颜色（同ell，仅支持三阶）
coll：coll就是coll，显示顶面十字及顶面角块（仅支持三阶）
ocell: 顶面角块及顶层十字（仅支持三阶）
wv：显示f2l和oll（仅支持三阶）
vh：显示f2l和顶面十字（仅支持三阶）
els: 显示f2l去掉最后一组角块，并显示顶面十字（仅支持三阶）
cls：显示f2l和oll（仅支持三阶）
cmll：桥氏解法中的第三步，显示左右桥以及顶层角块。（仅支持三阶）
cross:底层十字（仅支持三阶）
f2l_3: 显示FRD面的一组f2l（仅支持三阶）
f2l_2: 显示RDB和RDL面的两组f2l（仅支持三阶）
f2l_1:显示除RFD这一组f2l之外所有f2l（仅支持三阶）
f2l_sm: 显示RFD、BLD面的两个f2l（仅支持三阶）
f2b：显示桥氏左右桥（仅支持三阶）
line：无法形容，直接看图（仅支持三阶）
2x2x2：在RFD面显示2x2x2的颜色（仅支持三阶）
2x2x3:在RFLD面显示2x2x3的颜色（仅支持三阶）
默认值：.svg
网址方法格式：http://cube.cirder.co.uk/visualcube.png | .gif | .jpg | .svg | .tiff | .ico ?
例子：http://cube.cirder.co.uk/visualcube.png?bg=t
